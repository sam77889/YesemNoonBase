# NOON 洞察 · 用户使用手册

> 针对中东电商 NOON 平台的一站式数据抓取、清洗与可视化分析系统
> 适用版本：React 前端（玻璃拟态 / Bento 布局）+ FastAPI 后端 + 认证代理

---

## 一、系统简介

NOON 洞察帮助运营 / 选品人员从 NOON 平台批量抓取商品与评论数据，经过智能清洗后进入「追踪商品库」，并提供市场分析大盘、单品价格趋势、评论深度分析等可视化能力。

系统核心能力：

- **多源抓取**：付费代理（ScraperAPI / Oxylabs）与本地直连（curl_cffi Firefox 指纹，免费）两种引擎，自动清洗去重入库。
- **市场大盘**：价格段竞争热度、预估销量分布、价格-评论散点、品牌竞争力 TOP10，支持按类目联动。
- **追踪商品库**：全量已清洗商品，支持类目过滤、搜索、排序、批量剔除（软删除）。
- **单品价格趋势**：每个商品的历史价格 / 评分 / 评论数变化曲线。
- **深度评论分析**：单品 SKU 或整类目维度的评论情感、评分分布、优缺点、AI 摘要。

## 二、如何访问

系统通过认证代理对外提供服务，**所有页面与接口均需登录**。

| 环境 | 地址 | 说明 |
|------|------|------|
| 生产环境 | `https://noonbase.chen9527.top` | 经 Cloudflare Tunnel + 认证代理，公网可访问 |
| 本地开发 | `http://localhost:5173` | 前端 dev server（需另起后端 8000 / 8001） |

**登录步骤：**

1. 浏览器打开登录页（未登录访问任意地址会自动跳转 `/login`）。
2. 输入管理员用户名（`NOON_USER`）与密码（`NOON_PASS`）。
3. 登录成功后跳转至主控制台。

> ⚠️ 安全提示：登录失败次数限制为 **5 次 / 15 分钟**，超限后需等待 15 分钟再试。Session 有效期 30 分钟，超时需重新登录。
> 生产域名强制 HTTPS，仅经 Cloudflare 可正常登录；直接 HTTP 访问代理 IP 无法登录属正常现象。

## 四、功能模块详解

### 4.1 大盘总览（Overview）

进入系统默认页，展示当前「活跃追踪商品」的市场全景。

**类目过滤 Tab**
- 顶部 Tab 由成功抓取任务的类目**动态生成**（非静态分类），点击后所有图表联动过滤。
- 包含「全部」与「未分类」选项。
- 某类目下无商品时 Tab 会显示 0 或不出现。

**四大核心图表**

1. **蓝海价格段分析（组合图）**
   - 横轴为价格区间（自动分 20 桶，剔除异常离群值）。
   - 柱：该价格区间商品数（竞争热度）；线：该区间总评论数（需求热度）。
   - 用途：定位「商品少 + 评论多」的蓝海价格带。

2. **预估销量分布（面积图）**
   - 基于经验公式 `预估销量 = 评论数 × 38 + 1500 / max(价格,10) + 评分 × 20`（仅供可视化参考，非真实销量）。
   - 用途：判断市场是「赢家通吃」还是「长尾健康」。

3. **价格 vs 评论数（散点图）**
   - X 轴价格、Y 轴评论数，气泡大小 = 评分。
   - 用途：一眼定位「左上角」高评价、低价的潜力爆款。

4. **品牌竞争力 TOP10（横向柱状图）**
   - 按总评论声量排行，展示商品数、均分。
   - 用途：揭示类目品牌垄断格局。

> 注意：大盘统计**仅包含 ACTIVE 状态商品**，被剔除（软删除）的商品不计入。

---

### 4.2 付费搜查（Scraper）

适合大批量、稳定抓取，需要配置付费代理凭据（`SCRAPERAPI_KEY` / `OXYLABS_*`）。

**发起任务**

1. 在搜索框输入关键词（建议英文，如 `massage gun`）。中文关键词会自动翻译为英文再抓取。
2. 选择抓取数量（每批约 200 件）：200 / 600 / 1000 / 2000 件。
3. 选择抓取引擎：
   - **ScraperAPI**：默认，性价比高。
   - **Oxylabs**：更稳定，成本更高。
4. 点击「执行抓取」。

**任务状态**
- 下方「当前任务状态」卡片实时显示进度条与状态（处理中 / 成功 / 失败）。
- 成功（绿色满条）后数据自动清洗入库，类目 Tab 与大盘随之更新。
- 抓取过程中如需查看进度可切换到「系统日志」标签。

---

### 4.3 本地直搜（Fetcher）

🦊 基于 curl_cffi 模拟 Firefox TLS 指纹，直接绕过 NOON 的 Akamai 反爬，**无需任何付费 API**。

**使用方法**
1. 输入关键词（英文，如 `massage gun`）。
2. 选择抓取数量（200 / 600 / 1000 / 2000 件）。
3. 点击「直连抓取」。

**与付费搜查的区别**
- 免费、无配额限制，但抓取速度相对慢（逐商品并发抓取详情页，更全但更慢）。
- 任务归属以 `fetcher-` 前缀区分，本页只显示本地直搜任务。
- 适合日常轻量补数据、验证关键词、无代理凭据的环境。

> 说明：两种引擎抓回的数据都会进入同一商品库，大盘与数据库统一展示。

---

### 4.4 数据库（Database）

已清洗入库的全部商品列表。

**操作能力**
- **类目过滤**：顶部类目 Tab 与大盘联动。
- **关键词搜索**：按标题 / 品牌 / 类目模糊匹配。
- **排序**：点击表头按更新时间、价格、评分、评论数等排序（支持升 / 降序）。
- **分页**：支持调整每页条数，可承载千级数据量。
- **查看单品趋势**：点击任意一行 → 弹出「价格趋势弹窗」，展示该商品的价格 / 原价 / 折扣 / 评分 / 评论数历史曲线。
  - 弹窗内「最新数据」按钮：对该 SKU 触发一次增量抓取，刷新最新快照。
  - 「前往深度分析」按钮：跳转到深度分析页并自动带入该 SKU。
- **批量剔除**：勾选多个商品 → 点击批量删除。
  - 采用**软删除**（`status = INACTIVE`），历史价格快照保留可追溯，但不再计入大盘与统计。

---

### 4.5 深度分析（Analysis）

对评论数据进行情感与画像分析。支持两种模式（工具栏切换）：

**单品 SKU 模式**
1. 选择「单品 SKU」标签。
2. 输入 SKU（如 `N12345678A`，可在数据库页点击行获取）。
3. 点击「开始分析」。

**类目聚合模式**
1. 选择「类目聚合」标签。
2. 从类目 Tab 选择一个具体类目。
3. 点击「开始品类分析」（后台抓取该类目下所有商品的评论再聚合）。

**分析结果展示**
- 商品基础信息卡（单品模式）：名称、图片、SKU。
- **AI 摘要**：自动生成的综合评价摘要。
- **概览卡片**：评论数、综合评分、好评率等。
- **评分分布 / 情感饼图 / 优缺点条形 / 情感趋势 / 评论长度 / 可靠性 / 关键词 TOP15** 等多维图表。
- **近期评论列表**：展示代表性评论正文。
- **重新抓取最新**（单品模式，有数据时显示）：忽略本地缓存，强制实时抓取最新评论。

> 数据策略：优先读取已持久化的本地评论缓存并即时分析；无缓存或点击「重新抓取」时实时抓取。
> 已知限制：NOON 评论 API 受强反爬保护，单品可稳定获取的商品页 JSON-LD 中 3–5 条置顶评论为上限；完整评论（>5 条）需真实浏览器手动过 JS 挑战。部分商品页无评论数据时，会展示综合评分卡片而非报错。

---

### 4.6 系统日志（Logs）

集中展示抓取任务与分析任务的实时终端日志，带「打字机」滚动效果与颜色分级（成功 / 警告 / 错误 / 信息）。

- 顶部「清空」按钮可清除当前显示的日志块。
- 每个任务一个折叠块，点击展开查看完整日志。
- 用于排查抓取失败、代理报错、解析异常等问题。

## 五、典型工作流

**场景 A：从零调研一个新品类的市场**
1. 「付费搜查」或「本地直搜」输入关键词（如 `neck massager`），抓 600 件。
2. 等待任务成功 → 类目 Tab 出现新类目。
3. 「大盘总览」点击该类目，查看价格段 / 销量 / 品牌格局，定位蓝海价格带与潜力爆款。
4. 「数据库」搜索潜力商品，点击行查看价格趋势，剔除明显异常竞品。
5. 「深度分析」对目标 SKU 做评论分析，验证用户真实口碑。

**场景 B：持续监控已追踪商品**
1. 「数据库」浏览 / 排序，识别价格异常或评论暴涨的商品。
2. 点击行 → 「最新数据」刷新该 SKU 最新快照。
3. 「深度分析」→「类目聚合」定期复盘整类目口碑变化。

## 六、常见问题（FAQ）

**Q1：为什么大盘图表显示「暂无数据」？**
- 确认该类目下确有 ACTIVE 商品（被批量剔除的不计入）。
- 确认抓取任务已成功（非失败 / 处理中）。
- 类目过滤参数为内部英文类目值，前端已自动处理；若手动调接口，需传英文 `category` 值，勿传中文 label。

**Q2：抓取任务一直「处理中 / 加载动画」不结束？**
- 多页深抓可能耗时数分钟。前端会在任务到达终态（成功 / 失败）后自动停止动画。
- 若长时间无进展，到「系统日志」查看是否代理报错或触发反爬。

**Q3：本地直搜和付费搜查数据会冲突吗？**
- 不会。两者均按 SKU upsert 入库：商品不存在则新增，存在则更新基础信息，价格快照始终追加，历史完整保留。

**Q4：登录后页面空白 / 旧数据不更新？**
- 浏览器可能缓存了旧的前端 bundle。请**硬刷新**（Ctrl + Shift + R / Cmd + Shift + R）加载新版。

**Q5：评论分析提示「暂无评论」？**
- 该商品页可能本身无评论，或 JSON-LD 未内联评论（NOON 改版所致）。若显示了综合评分卡片，说明有评分但无正文可展示。

**Q6：登录频繁失败被锁定？**
- 连续 5 次错误密码会触发 15 分钟限流，请核对凭据后等待重试。

## 七、术语表

| 术语 | 含义 |
|------|------|
| **SKU** | NOON 商品唯一编号（如 `N12345678A`），抓取与分析的锚点 |
| **Provider** | 抓取引擎：`fetcher`（本地免费）/ `scraperapi` / `oxylabs`（付费） |
| **软删除** | 商品标记 `INACTIVE` 而非物理删除，保留历史快照、退出统计 |
| **价格快照** | 每次抓取为商品新增的一条时序记录（价格 / 原价 / 评分 / 评论数 / 折扣） |
| **类目 Tab** | 由成功任务类目动态生成的过滤标签，中文显示、内部映射英文值 |
| **蓝海价格带** | 商品竞争少但需求（评论）高的价格区间 |
| **预估销量** | 经验公式估算值，仅供可视化参考，非平台真实销量 |
| **Akamai** | NOON 使用的反爬防护，本地直搜通过 Firefox TLS 指纹绕过 |
